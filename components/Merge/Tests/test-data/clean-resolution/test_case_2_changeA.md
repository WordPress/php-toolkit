## 1. Merge Evaluation Test Suites and Strategies

The paper "Evaluation of Version Control Merge Tools" by the University of Washington provides a methodology for testing
merge tools. It adds a section on strategies.

It describes scenarios where:

- **Parent/Base Document**: The original document before any changes.
- **Change A**: A modification made in one branch.
- **Change B**: A separate modification made in another branch.
- **Merge Result**: The expected outcome after resolving conflicts.
    * **Note:**  Conflicts should be minimal.

The paper includes examples of merge scenarios, including code examples.
