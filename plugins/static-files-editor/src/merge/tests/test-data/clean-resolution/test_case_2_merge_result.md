
## 1. Merge Evaluation

The paper from the University of Washington provides a methodology for testing merge tools.  It adds a section on strategies.

It outlines these scenarios:

-   **Base Document**: The initial document.
-   **Change 1**: A modification.
-   **Change 2**: Another modification.
-   **Merged Result**: The expected final outcome.
    *   **Note:**  Conflicts should be minimal.

The paper has examples.  It also discusses limitations, including code examples.
